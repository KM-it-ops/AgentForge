# AgentForge v0.2 — Cursor adapter + platform-gap remediation report

Status: in progress (subagent-driven execution).
Branch: `main` (AgentForge convention; every emit creates a checkpoint).

## Goal

1. Add a fourth adapter for **Cursor**, matching the structure of `adapters/{claude-code,codex,generic}/`.
2. Integrate it into the round-trip CI, pack-install CI, and `npx agentforge` CLI.
3. Produce `docs/PLATFORM-GAPS.md` — a single audit of every documented platform gap across all adapters with a concrete remediation path for each.

## Constraints (do not violate)

- **Idempotency:** a second `node adapters/cursor/emit.js <target>` against an unchanged sandbox must leave `git status --porcelain` empty inside the sandbox.
- **Zero runtime deps:** the adapter must work with `miniYaml` fallback (no `js-yaml` install required), matching the other three adapters.
- **Cross-platform:** all shell scripts LF-only (enforced by `.gitattributes`). Adapter code must work on Linux + Windows runners (CI matrix: ubuntu-latest + windows-latest × node 20 + 22).
- **Spec is source of truth:** the adapter consumes `spec/*.yaml` from the repo root. It never duplicates spec values.

## Cursor platform model (reference)

Cursor's rule system as of 2026:
- `.cursorrules` — legacy single-file, project root, plain text. Still supported but deprecated.
- `.cursor/rules/*.mdc` — current modular format, YAML frontmatter + markdown body. Frontmatter fields:
  - `description: string` — human-readable summary used by Cursor's rule selection
  - `globs: string` (or list) — file glob(s) that activate the rule
  - `alwaysApply: bool` — when true, the rule is always in context

No skill-tool category. No lifecycle hooks. No native telemetry. No memory primitive. Same posture-portability constraints as the `generic` adapter, with the extra structure of `.mdc` rules.

## Tasks

### Task 1 — Build `adapters/cursor/`

Create a new adapter directory mirroring `adapters/generic/` in shape:

```
adapters/cursor/
├── README.md                       # what it emits, idempotency note, platform-gap list
├── emit.js                         # entry point — node emit.js <target-dir>
├── templates/
│   ├── cursorrules.tmpl            # legacy single-file body
│   ├── rule-identity.mdc.tmpl      # alwaysApply: true — identity + execution rules + memory pointer
│   ├── rule-router.mdc.tmpl        # alwaysApply: true — skill router table (rendered from spec.router)
│   └── rule-route.mdc.tmpl         # per-route rule with description + globs
└── scripts/
    └── dead-skills-report.sh       # ported verbatim from adapters/generic/scripts/
```

The emitter must:

1. Take exactly one CLI arg — `<target-dir>` — and resolve `~` to `$HOME`.
2. `ensureDir(target)` then run a pre-emit git checkpoint (best-effort: skip if not a git repo).
3. Load `spec/{identity,router,memory,automation,telemetry}.yaml` via the same `parseYaml` / `miniYaml` fallback the generic adapter uses (copy the pattern, do not import — adapters stay standalone).
4. Render and write the following files (via `writeIfChanged` — only touch disk when content differs):
   - `<target>/.cursorrules` — consolidated legacy view: identity block + execution rules + skill router table + memory protocol pointer.
   - `<target>/.cursor/rules/identity.mdc` — frontmatter `alwaysApply: true`, body = identity + execution rules.
   - `<target>/.cursor/rules/router.mdc` — frontmatter `alwaysApply: true`, body = full router table from `spec.router.routes`.
   - `<target>/.cursor/rules/route-<key>.mdc` per route in `spec.router.routes` — frontmatter `description: <human-readable>`, `globs: "**/*"` (default), `alwaysApply: false`. Body explains when to load the target skill. (One rule per route.)
   - `<target>/MEMORY.md` — same memory-index seed the generic adapter writes.
   - `<target>/memory/user/.gitkeep`, `memory/feedback/.gitkeep`, `memory/project/.gitkeep`, `memory/reference/.gitkeep`.
   - `<target>/memory/feedback/session-log.md` — seeded per `spec.memory` (match generic adapter behavior).
   - `<target>/skills/README.md` — explains the manual-add convention for SKILL.md-prefixed dirs.
   - `<target>/telemetry/README.md` — honest gap doc (Cursor has no telemetry primitive).
   - `<target>/scripts/dead-skills-report.sh` — verbatim copy from `adapters/generic/scripts/dead-skills-report.sh`.
5. Emit a JSON summary to stdout in the same shape as the generic adapter:
   ```json
   { "target": "...", "checkpoint_sha": "...", "files_written": <n>, "files_changed": <n>, "details": [...] }
   ```
6. Be **idempotent**: running it twice against the same target must produce `files_changed: 0` on the second run and leave `git status --porcelain` empty.

Working directory for the subagent: `C:/AI/AgentForge`.

**Reference implementations to read before starting:**
- `adapters/generic/emit.js` — closest shape (no hooks, no automation).
- `adapters/codex/emit.js` — template rendering, router-table generation from `spec.router`.
- `spec/router.yaml` and `spec/identity.yaml` — input data.

**Do NOT:**
- Add a `package.json` or any npm dependency. The adapter must work standalone.
- Invent fields not present in the spec. If a Cursor-specific value is needed, hardcode a sensible default and call it out in `adapters/cursor/README.md` under "platform gaps".
- Copy template files verbatim from another adapter without verifying the variables resolve from the spec.

**Acceptance:**
- `node adapters/cursor/emit.js /tmp/cursor-test` succeeds and writes ≥ 12 files.
- Second run produces `"files_changed": 0` and leaves `git -C /tmp/cursor-test status --porcelain` empty.
- The emitter handles a target dir that is not a git repo gracefully (skip checkpoint with a log line).
- All shell scripts LF-only.

### Task 2 — Wire Cursor into round-trip, pack-install, and the CLI

Extend three files to include `cursor` as a first-class adapter:

1. `scripts/round-trip-test.sh` — append `"cursor"` to the `ADAPTERS=(...)` array. The script's idempotency check already handles JSON-summary adapters (it greps for `"files_changed": 0`).
2. `scripts/pack-install-test.sh` — append `"cursor"` to the iteration list and add an entry to `EXPECTED_FILES[cursor]=<n>` matching Task 1's actual emitted file count.
3. `bin/agentforge.js` — add `cursor` to the `ADAPTERS` object. Cursor has no canonical install directory (analogous to `generic`); set `defaultDir: null` so `--dir` is required.

Update `package.json` `files:` only if Task 1 added a directory not already covered by the existing `adapters/` glob (it shouldn't — `adapters/` covers everything).

**Acceptance:**
- `bash scripts/round-trip-test.sh` passes locally with all four adapters green.
- `bash scripts/pack-install-test.sh` passes locally.
- `node bin/agentforge.js init cursor` (no `--dir`) prints a clear error pointing at `--dir`.
- `node bin/agentforge.js init cursor --dir /tmp/cursor-cli-test` succeeds and produces the same file count as a direct `emit.js` call.

### Task 3 — Produce `docs/PLATFORM-GAPS.md`

Research-only. No code changes. Produces one markdown report at `docs/PLATFORM-GAPS.md`.

Audit every documented platform gap across all four adapters and propose a concrete remediation path for each. Sources to read:

- `docs/VERIFICATION-v0.1.0.md` — 6 documented Codex gaps + generic adapter coverage notes.
- `adapters/codex/README.md` — fuller gap descriptions.
- `adapters/generic/README.md` — generic adapter's intentional gap surface.
- `adapters/cursor/README.md` — gaps surfaced by Task 1.
- `adapters/cursor/emit.js` — any TODOs or comments flagging Cursor-specific limitations.

For each gap, the report must include:

| Field | Content |
|---|---|
| Adapter | claude-code \| codex \| generic \| cursor |
| Gap ID | stable kebab-case identifier (e.g. `codex-notify-coarse`) |
| Description | what doesn't work, in 1-2 sentences |
| User impact | what breaks for the user, by severity (`blocker` \| `degraded` \| `cosmetic`) |
| Root cause | platform limitation vs. adapter implementation choice |
| Remediation | one of: `workaround-now` (concrete workaround the user applies today), `adapter-fix` (something the AgentForge codebase can ship), `upstream-only` (only fixable by the platform vendor), `docs-only` (acceptable as-is, document better) |
| Concrete next step | exact action: file path + change, or upstream link to file, or doc paragraph to write |
| Effort | `S` (≤1h) \| `M` (≤4h) \| `L` (>4h) |
| Priority | `P0` (ship now) \| `P1` (v0.2) \| `P2` (v0.3+) |

End the report with a "**Recommended v0.2 cleanup batch**" section listing the P0 + P1 items grouped by adapter, with a one-paragraph rationale for the cut line.

**Acceptance:**
- File exists at `docs/PLATFORM-GAPS.md`.
- All gaps from `docs/VERIFICATION-v0.1.0.md` Codex section (6 items) and the generic adapter section appear, plus every gap surfaced in `adapters/cursor/README.md` from Task 1.
- Every row has a `Concrete next step` with a file path or external link. No vague "investigate further" entries.
- Report is internally consistent: every `adapter-fix` row's next step points at a real file that exists in the repo.

## Workflow

Tasks run serially in the order listed. After each task:

1. Implementer self-reviews and commits with a conventional-commit message.
2. Spec compliance review (separate subagent) verifies the commit matches the task spec verbatim.
3. Code quality review (separate subagent) verifies the implementation is clean and tested.
4. Only after both reviews pass does the next task begin.

After all three tasks, a final code-reviewer subagent reviews the full delta against `main` at task-1 base.
